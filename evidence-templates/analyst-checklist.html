First Five Minutes

Page 4 is not about perfect certainty. It is about whether the analyst can move from first alert to defensible action using the same evidence chain the dashboard presents.

1

Confirm Entity

Lock the user, workload, host, or service account before pivoting so the same actor remains visible across every page.

2

Measure Novelty

Check whether the source, geography, API pattern, or process chain is new enough to justify deeper handling.

3

Preserve Proof

Bookmark the strongest event, rule, or audit artifact before chasing additional context so the core story is never lost.

4

Estimate Blast Radius

Use asset criticality, privileged group changes, or data-store access to decide whether this is a queue item or an incident.

5

Choose a Bias

Decide whether the next action should bias toward containment, validation, or escalation based on business exposure, not just alert severity.

6

Assign the Next Move

End the first five minutes with a named owner, a concrete next proof target, and an explicit time expectation.

Operator Standard The workflow is successful only if a second analyst could continue the case from the captured proof without relying on unwritten verbal context.
Escalate Early When Identity privilege change, secret retrieval, database access, or sensitive bucket activity follows the first signal.
